General info

This GUI demonstrates how this boilerplate is handling errors with $http and $sailsSocket responses. Note that this magic is done automatic, so you don't have to do anything extra for this functionality. Actual functionality is done via angular interceptor, so all of this is attached automatically to all $http and $sailsSocket communications.

This will intercept all errors from $http and $sailsSocket requests. Note that shown message depends about actual error response from server. Errors are shown to user via Message service which uses noty2 library. You can also use this Message service to show desired messaged to users if you like to.

Frontend

  • messages.js (module configuration)
  • messages-controller.js (controller)
  • messages.html (template)
  • ErrorInterceptor.js (error interceptor)
  • Message.js (message service)
